Resumo da proposta

Pitch do jogo

Blink, um fantasminha digital, precisa coletar Data Cores em um laboratorio sci-fi para hackear o mainframe e escapar antes de ser deletado.

Platformer 2D Unity 6 ESP32 + OLED Narrativa leve Evento de 1 dia

Diferencial

O projeto mistura jogo e hardware fisico: o Unity envia reacoes para o ESP32, e o ESP32 devolve comandos de controle. O OLED funciona como uma extensao emocional do personagem.

Area Esperado pela proposta Fonte principal
Gameplay Andar, pular, coletar 3 itens, vencer ao coletar tudo ou chegar ao portal README.md, requirements RF-01 a RF-04
Hardware ESP32 com OLED lendo I, T, D e enviando L1/L0/R1/R0/U requirements RF-05 e RF-06
Visual Laboratorio futurista, Blink, Data Cores, laser, drone e acido storytelling e design visual
UX HUD com contagem de coletaveis e painel de vitoria requirements RF-07

Arquitetura pretendida

Unity

  • PlayerController controla Blink
  • GameManager conta progresso e mostra vitoria
  • Collectible dispara coleta
  • LevelEnd fecha a fase

Serial

  • ESP32SerialReader faz leitura em thread separada
  • Fila concorrente leva eventos para a main thread
  • Jogo nao deve travar se a serial falhar

ESP32

  • Recebe 1 char para trocar a expressao no OLED
  • Le botao fisico e manda eventos de direcao e pulo
  • Baud rate definido em 115200
Fluxo central da proposta: jogadora age no jogo → Unity detecta → envia pela USB → ESP32 muda a carinha do Blink no OLED.

Leitura por camada

Narrativa e proposta

Muito consistentes. A historia, os elementos de mundo, a tabela de expressoes do OLED e o escopo minimo convergem bem entre README, storytelling e spec tecnica.

Codigo

O codigo segue um padrao simples e adequado para evento curto: singleton para servicos centrais, trigger para itens, fallback de input e tratamento gracioso para falha de serial. Agora ele tambem cobre vidas, HUD de Polvinas e fechamento minimo da cena demo.

Cenas

A estrutura de cenas agora esta coerente: Ecto-Scape e a cena jogavel oficial, enquanto SceneAutoSetup garante o setup minimo em runtime e SampleScene fica apenas como template fora do build.

Operacao do evento

O repositrio foi pensado tambem como material pedagogico. Isso explica a grande quantidade de guias e templates, mas gera alguns pontos onde a documentacao esta mais madura do que a entrega executavel.

Riscos principais

Risco 1

A base funcional esta pronta, mas ainda existe espaco para substituir o bootstrap por uma montagem 100% manual no layout final.

Risco 2

O bootstrap resolve o minimo funcional, mas a fase ainda pode ganhar um layout mais intencional quando for aberta no Unity Editor.

Risco 3

Os testes cobrem regras centrais, mas ainda faltou validar tudo em runtime dentro do Unity Editor e no hardware real.
